GUIDE D'INSTALLATION

CRYPTO ORACLE v2 — APP MOBILE + ALERTES EMAIL · GRATUIT SANS CODE

FICHIERS À DÉPLOYER

index.htmlLe dashboard principal (renommé depuis crypto-oracle-v2.html)
manifest.jsonManifeste PWA — permet l'installation sur écran d'accueil
sw.jsService Worker — permet l'utilisation hors connexion
icon-192.svgIcône de l'app sur l'écran d'accueil
alerts.jsonFlux d'alertes — surveillé par Make.com pour les emails
Important : Tous ces fichiers doivent être dans le même dossier avant le déploiement.

PARTIE 1 — APP MOBILE VIA NETLIFY DROP

ÉTAPE 01
Déposer les fichiers sur Netlify
Allez sur netlify.com/drop (pas besoin de compte).
Créez un dossier sur votre bureau, mettez-y les 5 fichiers ci-dessus.
Glissez-déposez le dossier entier dans la zone bleue de Netlify Drop.
En 10 secondes vous obtenez une URL du type :
https://wonderful-name-12345.netlify.app
Notez cette URL, vous en aurez besoin pour Make.com.
ÉTAPE 02 — Sur iPhone
Ajouter à l'écran d'accueil (iOS)
1. Ouvrez Safari (pas Chrome, pas Firefox — uniquement Safari pour iOS)
2. Allez sur votre URL Netlify
3. Appuyez sur l'icône Partager (carré avec flèche vers le haut)
4. Faites défiler et tapez "Sur l'écran d'accueil"
5. Nommez l'app "Crypto Oracle" → Ajouter

L'app apparaît sur votre écran d'accueil avec l'icône, s'ouvre en plein écran sans barre Safari.
ÉTAPE 02 — Sur Android
Ajouter à l'écran d'accueil (Android)
1. Ouvrez Chrome
2. Allez sur votre URL Netlify
3. Tapez les 3 points en haut à droite
4. "Ajouter à l'écran d'accueil"
5. Confirmer → Ajouter

Chrome peut même afficher une bannière "Installer l'application" automatiquement.
Pour mettre à jour le dashboard (nouvelles données) : redemandez à Claude de régénérer les fichiers, puis revenez sur netlify.com/drop et redéposez le dossier mis à jour. L'URL reste la même si vous utilisez le même site Netlify (créez un compte gratuit pour garder la même URL).

PARTIE 2 — ALERTES EMAIL VIA MAKE.COM

ÉTAPE 01
Créer un compte Make.com gratuit
Allez sur make.com → Sign up free.
Le plan gratuit inclut 1000 opérations/mois — largement suffisant pour des vérifications horaires.
Une vérification par heure = 24/jour = 720/mois.
ÉTAPE 02
Créer un nouveau scénario
Dans Make.com : Create a new scenario
Cliquez sur le + pour ajouter le premier module.
ÉTAPE 03
Module 1 — Déclencheur : Schedule
Cherchez "Schedule" → sélectionnez-le comme déclencheur.
Configurez :
⏱Run scenarioEvery 1 hour
📅DaysEvery day
ÉTAPE 04
Module 2 — Récupérer le fichier alerts.json
Ajoutez un module "HTTP" → "Make a request".
Configurez :
🌐URLhttps://VOTRE-SITE.netlify.app/alerts.json
📋MethodGET
📦Parse responseYes (JSON)
ÉTAPE 05
Module 3 — Filtre : Score ≥ 80 ET alerte EXTREME
Entre le module HTTP et le module Email, ajoutez un Filter (l'entonnoir entre deux modules).
Condition :
🔢summary.alerts_extremeGreater than: 0
→ Le scénario continue uniquement si au moins une alerte EXTREME est présente.
ÉTAPE 06
Module 4 — Envoyer l'email
Ajoutez le module "Email" → "Send an email" (ou Gmail si vous préférez).
Configurez :
📧Tovotre@email.com
📌Subject🚨 CRYPTO ORACLE — Alerte EXTREME détectée
📝ContentVoir ci-dessous
Dans le corps du message, utilisez les variables Make.com :

🚨 ALERTE EXTRÊME — Crypto Oracle v2

Top coin : {{summary.top_coin}}
Score : {{summary.top_score}} / 100
Alertes extrêmes : {{summary.alerts_extreme}}
Fear & Greed : {{market_context.fear_greed}}

Ouvrir le dashboard : https://VOTRE-SITE.netlify.app
ÉTAPE 07
Activer le scénario
Cliquez sur Save puis activez le toggle en bas à gauche.
Cliquez "Run once" pour tester immédiatement.
Si tout est vert → le scénario tourne automatiquement toutes les heures.
Pour des alertes encore plus urgentes : Make.com supporte aussi les notifications push (via Pushover, Pushbullet) et les SMS (via Twilio). Même logique, module différent à l'étape 6.
